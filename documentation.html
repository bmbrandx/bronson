Introduction

Bronson was borne from desire to build a highly decoupled, testable, scalable interoperable modules on top of the greatness that is RequireJS. Bronson is heavily inspired by the work of Nicholas Zaka's talk on scalable javascript as well as Addy Osmani's articles and frameworks. At it's core Bronson & RequireJS free you to build sophisticated modules without the trappings of complicated woven interdepdencies.

Features

  • Advanced publish/subscribe framework
  • Module loading/unloading sandboxing
  • Base Module for bootstrapping mini applications

Dependencies

  • JQuery
  • RequireJS

Api

The API layer is the public programming interface for communicating with R8. The API abstracts the Core and Permissions layer and presents a closed to modification interface.

Overview

  • Provide interface to pass module loading to core
  • Coordinate with Permissions layer to validate events
  • Provide interface to pass publish/subscribe events to core

createModule()

The createModule loads a module via RequireJS and initializes it. Your module should bootstrap your mini application.

Simple Example {% highlight coffeescript %} require ['bronson'], (Bronson) -> Bronson.Api.createModule 'example/path/sampleModule', -> # Module has been created {% endhighlight %} Example with Configuration Object

It is also possible to pass configuration object during the construction of the module. This configuration object will be passed to the constructor your module bootstrapper when you extend Bronson.Module

{% highlight coffeescript %} require ['bronson'], (Bronson) -> # Sample data to be passed to module configurationObject = el: 'div#modules' url: 'http://example.com/api/data.json' Bronson.Api.createModule 'example/path/sampleModule', configurationObject, -> # Module has been created {% endhighlight %}

stopModule()

Unloads the artifiacts loaded in when createModule was initially called.

{% highlight coffeescript %} require ['bronson'], (Bronson) -> Bronson.Api.stopModule 'example/path/sampleModule', -> # Module has been stopped {% endhighlight %}

subscribe

Methods

  • api.publish(module, event, data)
  • api.subscribe(event, data, callback)
  • api.unsubscribe(event)

Examples

Simple Publish
{% highlight coffeescript %} # With AMD require ['bronson'], (Bronson) -> Bronson.Core.publish 'TestEvent' # Without Bronson.Core.publish 'TestEvent' {% endhighlight %}
Simple Subscribing
{% highlight coffeescript %} # With AMD require ['bronson'], (Bronson) -> Bronson.Core.subscribe 'TestModule', 'TestEvent', (msg) -> console.log 'successfully subscribe' # Without Bronson.Core.subscribe 'TestModule', 'TestEvent', (msg) -> console.log 'sucessfully subscribed' {% endhighlight %}
Publishing with Data
{% highlight coffeescript %} # With AMD require ['bronson'], (Bronson) -> Bronson.Core.publish 'TestEvent', { foo: 'bar' } (msg) -> console.log 'successfully published' {% endhighlight %}
Sending data to a subscriber
{% highlight coffeescript %} # With AMD require ['bronson'], (Bronson) -> Bronson.Core.subscribe 'TestModule', 'TestEvent', (msg) -> console.log 'thanks for the #{msg}' Bronson.Core.publish 'TestEvent', { foo: 'bar' } (msg) -> console.log 'successfully published' {% endhighlight %}
