Introduction

Bronson is an opinonated application framework comprising of the tools necessary to build large scalable web applications as outlined by Nicholas Zakas and Addy Osmani. Bronson sit's on top of RequireJS & JQuery offering libraries for the loading, unloading, messaging and bootstraping of modules. These modules in turn acts as small loosely coupled applications in the context of a large appication. By building our larger applications as a series of smaller applications we greatly increase the performance, testability and maintainability of our code base.

Acknowledgements

  • Nicholas Zakas: Scalable Javascript Application Architecture
  • Addy Osmani: Patterns For Large-Scale JavaScript Application Architecture

Features

  • Module loading & unloading through Bronson.Core(mediator pattern)
  • Univeral interface offered through Bronson.Api(fascade pattern)
  • Managed publish & subscribing of events through Bronson.Core & Bronson.Permissions

Dependencies

  • jQuery
  • RequireJS

Api

Bronson.Api acts as the Fascade in Bronson and is the public programming interface for communicating with Bronson. The Api abstracts the Bronson.Core & Bronson.Permission layer's presenting a closed to modification interface.

Overview

  • Provide interface to pass module loading to core
  • Coordinate with Permissions layer to validate events
  • Provide interface to pass publish/subscribe events to core

createModule

The createModule loads a module via RequireJS and initializes it. Your implemented module should bootstrap your mini application.

Simple Example

{% highlight coffeescript %} require ['bronson'], (Bronson) -> Bronson.Api.createModule 'example/path/sampleModule', -> # Module has been created {% endhighlight %}

Example with Configuration Object

It is also possible to pass the configuration object during the construction of the module. This configuration object will be passed to the constructor of your module bootstrapper when you extend Bronson.Module.

{% highlight coffeescript %} require ['bronson'], (Bronson) -> # Sample data to be passed to module configurationObject = el: 'div#modules' url: 'http://example.com/api/data.json' Bronson.Api.createModule 'example/path/sampleModule', configurationObject, -> # Module has been created {% endhighlight %}

stopModule

Unloads the artifiacts loaded in when createModule was initially called.

{% highlight coffeescript %} require ['bronson'], (Bronson) -> Bronson.Api.stopModule 'example/path/sampleModule', -> # Module has been stopped {% endhighlight %}

stopAllModules

Unloads the artifiacts loaded for all created modules calling stopModule on each in turn.

{% highlight coffeescript %} require ['bronson'], (Bronson) -> Bronson.Api.stopAllModules -> # All modules have been stopped {% endhighlight %}

subscribe

Subscribe a module to an event and if Bronson.Permission is enabled validate the event.

Simple Example

{% highlight coffeescript %} require ['bronson'], (Bronson) -> Bronson.Api.subscribe 'example/path/sampleModule', 'TestEvent', -> # Module subscribed {% endhighlight %}

Example recieving data

{% highlight coffeescript %} require ['bronson'], (Bronson) -> Bronson.Api.subscribe 'example/path/sampleModule', 'TestEvent', (data) -> # Event triggered with data console.log data {% endhighlight %}

publish

Publishes an event to all subscribers

Simple Example

{% highlight coffeescript %} require ['bronson'], (Bronson) -> Bronson.Api.publish 'TestEvent', -> # Event published {% endhighlight %}

Example with data

It's also possible to pass data along with the event

{% highlight coffeescript %} require ['bronson'], (Bronson) -> Bronson.Api.publish 'TestEvent', { foo: 'bar' } -> # Event published {% endhighlight %}

unsubscribe

Unsubscribes an event from a module

{% highlight coffeescript %} require ['bronson'], (Bronson) -> Bronson.Api.unsubscribe 'example/path/sampleModule', 'TestEvent', -> # Event unsubscribed {% endhighlight %}

unsubscribeAll

Unsubscribes all events from a module

{% highlight coffeescript %} require ['bronson'], (Bronson) -> Bronson.Api.unsubscribeAll 'example/path/sampleModule', -> # All events unsubscribed {% endhighlight %}
