Api

Bronson.Api acts as the Fascade in Bronson and is the public programming interface for communicating with Bronson. The Api abstracts the Bronson.Core & Bronson.Permission layer's presenting a closed to modification interface.

Overview

  • Provide interface to pass module loading to core
  • Coordinate with Permissions layer to validate events
  • Provide interface to pass publish/subscribe events to core

loadModule

Loads a module via RequireJS and calls your modules load method.

Simple Example

{% highlight coffeescript %} require ['bronson'], (Bronson) -> Bronson.Api.loadModule 'example/path/sampleModule', -> # Module has been created {% endhighlight %}

Example with Configuration Object

It is also possible to pass the configuration object during the construction of the module. This configuration object will be passed to the constructor of your module bootstrapper when you extend Bronson.Module.

{% highlight coffeescript %} require ['bronson'], (Bronson) -> # Sample data to be passed to module configurationObject = el: 'div#modules' url: 'http://example.com/api/data.json' Bronson.Api.loadModule 'example/path/sampleModule', (-> # module loaded ), configurationObject # The configuration object to pass to the module , false # Whether or not to autostart the module {% endhighlight %}

unloadModule

Unloads a module. Call's the unload method of the module whose instance id you provide. Typically used to clean up memory.

{% highlight coffeescript %} require ['bronson'], (Bronson) -> Bronson.Api.unloadModule module.id {% endhighlight %}

unloadAllModules

Calls unload method of every instanced module.

{% highlight coffeescript %} require ['bronson'], (Bronson) -> Bronson.Api.unloadAllModules {% endhighlight %}

stopModule

Stops a module. Call's the stop method of the module whose instance id you provide. Typically used when you want to remove an item from the DOM but keep it in memory.

{% highlight coffeescript %} require ['bronson'], (Bronson) -> Bronson.Api.stopModule module.id {% endhighlight %}

stopAllModules

Unloads the artifiacts loaded for all created modules calling stopModule on each in turn.

{% highlight coffeescript %} require ['bronson'], (Bronson) -> Bronson.Api.stopAllModules -> # All modules have been stopped {% endhighlight %}

subscribe

Subscribe a module to an event and if Bronson.Permission is enabled validate the event.

Simple Example

{% highlight coffeescript %} require ['bronson'], (Bronson) -> Bronson.Api.subscribe 'TestSubscriber', 'TestEvent', -> # Module subscribed {% endhighlight %}

Example recieving data

{% highlight coffeescript %} require ['bronson'], (Bronson) -> Bronson.Api.subscribe 'TestSubscriber', 'TestEvent', (data) -> # Event triggered with data console.log data {% endhighlight %}

publish

Publishes an event to all subscribers

Simple Example

{% highlight coffeescript %} require ['bronson'], (Bronson) -> Bronson.Api.publish 'TestEvent', -> # Event published {% endhighlight %}

Example with data

It's also possible to pass data along with the event

{% highlight coffeescript %} require ['bronson'], (Bronson) -> Bronson.Api.publish 'TestEvent', { foo: 'bar' } -> # Event published {% endhighlight %}

unsubscribe

Unsubscribes an event from a module

{% highlight coffeescript %} require ['bronson'], (Bronson) -> Bronson.Api.unsubscribe 'example/path/sampleModule', 'TestEvent', -> # Event unsubscribed {% endhighlight %}

unsubscribeAll

Unsubscribes all events from a module

{% highlight coffeescript %} require ['bronson'], (Bronson) -> Bronson.Api.unsubscribeAll 'example/path/sampleModule', -> # All events unsubscribed {% endhighlight %}
